# Firefly UX Design Document

## Overview

Firefly is a **unified mobile application** that combines customer ordering, vendor management, and delivery coordination into a single chat-centric experience. The design philosophy prioritizes relationships and communication over traditional e-commerce patterns.

---

## Core Design Principles

### 1. Unified App with Role Switching
Rather than separate apps for customers, vendors, and delivery drivers, Firefly uses a single app where users switch between roles. This reflects the reality that in a community co-op, the same person might order food one day and make deliveries the next.

### 2. Relationship-First Design
The home screen displays past interactions like a messaging app, not a storefront. This emphasizes ongoing relationships within the community rather than transactional commerce.

### 3. Chat-Centric Ordering
Orders don't happen through a traditional checkout flowâ€”they happen inside group chats. This enables natural communication between customers, vendors, and delivery drivers.

### 4. Community Activity Feed
An optional public feed shows community activity (like an MMORPG), fostering connection and transparency within the co-op.

## Accessibility Considerations

1. **Color is not the only indicator** - Role colors are supplemented with icons and labels
2. **Large tap targets** - Minimum 44x44pt for all interactive elements
3. **Screen reader support** - All images have alt text, status updates are announced
4. **Reduced motion** - Respect system settings for animations
5. **High contrast mode** - Support system high contrast settings

---

## Future Considerations

1. **Multi-language support** - Interface strings are externalized for i18n
2. **Dark mode** - Color palette designed with dark variants
3. **Tablet/Desktop** - Responsive layouts for larger screens
4. **Offline support** - Queue actions when offline, sync when connected
5. **Accessibility audit** - WCAG 2.1 AA compliance target
